# 常见问题排查指南

## 错误：webapi_getwxaasyncsecinfo:fail access_token missing (41001)

### 问题说明
这个错误是微信小游戏开发者工具内部API调用失败导致的，**不影响游戏正常运行**。

### 原因分析
1. 微信开发者工具版本问题
2. 项目在本地开发环境运行，某些需要服务器权限的API不可用
3. 微信内部安全检查API调用失败

### 解决方案

#### 方案1：忽略该错误（推荐）
该错误已被错误处理器捕获，不会影响游戏运行。你可以：
- 继续开发游戏
- 忽略控制台中的该错误信息

#### 方案2：更新开发者工具
1. 检查微信开发者工具版本
2. 更新到最新稳定版
   - 菜单栏 → 帮助 → 检查更新
   - 或下载最新版本：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

#### 方案3：检查项目配置
确保 `project.config.json` 中的 `appid` 正确：
```json
{
  "appid": "wx1797da1097582e92"
}
```

#### 方案4：清理缓存
1. 微信开发者工具菜单栏 → 项目 → 清缓存
2. 选择"清除全部缓存"
3. 重新编译项目

### 验证游戏运行正常
如果看到以下日志，说明游戏运行正常：
```
[喵星奇缘] 游戏启动完成
[Game] 开始加载资源...
[Game] 资源加载完成
[Field] 遭遇敌人: wild_cat {id: 'wild_cat', name: '野猫', ...}
```

## 其他常见错误

### 图片加载失败
**错误信息：** `[AssetManager] 加载失败: images/xxx.png`

**解决方案：**
1. 检查图片路径是否正确
2. 检查图片文件是否存在
3. 检查图片格式（支持 png, jpg, gif）

### 场景切换失败
**错误信息：** `TypeError: Cannot read property 'init' of null`

**解决方案：**
1. 检查场景名称是否正确
2. 检查场景类是否正确导入
3. 查看控制台日志定位问题

### 触摸事件无效
**可能原因：**
1. 触摸区域被其他UI遮挡
2. 事件监听器未正确注册
3. 坐标转换错误

**调试方法：**
在触摸事件处理函数中添加日志：
```javascript
console.log('[Touch] x:', x, 'y:', y)
```

## 开发建议

### 1. 使用版本控制
定期提交代码到Git，方便回滚：
```bash
git add .
git commit -m "feat: 添加新功能"
git push
```

### 2. 定期测试
- 测试各个场景切换
- 测试战斗流程
- 测试资源加载
- 测试触摸交互

### 3. 查看日志
使用控制台日志调试：
```javascript
console.log('[标签] 描述信息', 数据)
```

### 4. 性能优化
- 使用对象池减少内存分配
- 优化渲染逻辑，只绘制可见元素
- 控制动画帧率
